Dell Rewards Enrollment Experience

I designed a conversion-focused landing page that simplified the Dell Rewards value proposition, improved scanability, and guided users toward enrollment through stronger hierarchy and structure.

Conversion-focused structure
Clearer rewards communication
Responsive enrollment flow
Objective

Create a high-converting Dell Rewards landing page that made the program easy to understand, easy to scan, and easy to act on across desktop and mobile.

Role

UI Designer
Landing page design

Team
Me PM DV
Timeframe

3 weeks

Overview

Dell Rewards needed a landing page that could explain several benefits quickly: rewards points, member offers, financing-related value, and an easy path to enrollment. Unlike a redesign project, there was not an existing page to critique. The challenge was to build the experience from the ground up using strong landing page structure, clear hierarchy, and a guided flow that helped users understand the value without feeling overwhelmed.

I approached the page as an acquisition experience, not just a visual design. The goal was to make the offer easy to understand from the first screen, organize the supporting content into scannable sections, and repeat the enrollment action in places where users would naturally be ready to move forward.

Design Challenge

The main challenge was translating a feature-rich rewards program into a simple, benefit-led story. The page needed to feel like Dell, communicate value quickly, and create a clear path toward sign-up without overloading users with too much information at once.

User needs

  • Understand what Dell Rewards offers without digging through dense copy
  • See the value of joining early in the experience
  • Compare benefits, rewards, and member perks quickly
  • Trust the offer because it is attached to a major technology brand
  • Find a clear enrollment path on desktop and mobile

Design considerations

  • Balance rewards messaging, promotional offers, and program education
  • Create a strong above-the-fold value proposition
  • Keep CTA placement visible without making the page feel pushy
  • Use modular sections so the page stayed easy to scan
  • Maintain Dell brand trust through color, imagery, and visual consistency

Structuring the Enrollment Journey

Because the page needed to communicate multiple benefits, I focused heavily on section order and pacing. The structure was designed to answer the user's questions in sequence: what is the program, why should I join, how do rewards work, and what should I do next?

The flow gave each section a specific job. The hero introduced the core value. The steps section explained how the program works. The earning points section made the reward mechanics easier to understand. Member perks created a stronger reason to join, and the FAQ helped reduce hesitation before enrollment.

Visual Hierarchy & Scanability

The page was designed around fast comprehension. Instead of forcing users to read every section in detail, I used large headlines, clear content groupings, product-forward imagery, and consistent CTA placement to help users move through the page with less effort.

Solution

I built the experience around a direct rewards story: understand the value, see how it works, explore the perks, and enroll. Every major design decision supported clarity, trust, or forward momentum.

How the experience was structured

  • I prioritized the Dell Rewards value proposition above the fold
  • I used product imagery to make the experience feel more tangible and brand-aligned
  • I organized the program details into clear, scannable modules
  • I created a stronger CTA hierarchy so enrollment always felt like the primary action
  • I broke the rewards explanation into step-by-step sections
  • I used visual pacing to keep the page from feeling too dense
  • I carried Dell's blue palette and clean design language through the page
  • I made the layout responsive so the enrollment path stayed clear on mobile
The hero leads with a clear rewards promise and places the primary enrollment action near the first value message.

Conversion-Focused Design Decisions

Without a prior page to compare against, the work centered on building the right structure from the start. These decisions helped turn a feature-rich rewards program into a clearer enrollment experience.

Design decision

  • Benefit-led hero with direct rewards messaging
  • Repeated enrollment CTA placement
  • Step-by-step rewards explanation
  • Modular benefit and perk sections
  • FAQ positioned near final decision points
  • Responsive layouts planned from the start

Conversion intent

  • Help users understand the offer faster
  • Reduce friction when users are ready to join
  • Make the program mechanics easier to follow
  • Improve scanability and reduce cognitive load
  • Answer objections before they become drop-off points
  • Keep the enrollment path clear on smaller screens

Mobile Experience Walkthrough

The mobile version keeps the same story, but simplifies the pacing for a vertical scroll. The goal was to preserve the value proposition, benefit sections, FAQs, and enrollment actions without making the page feel crowded.

Dell Rewards Landing Page Outcome

The final experience created a clearer path through the Dell Rewards enrollment journey by simplifying how benefits, rewards points, member perks, and program details were presented. By improving hierarchy, scanability, and CTA consistency, the page was designed to help users understand the offer faster and move through the experience with less friction across desktop and mobile.

What improved

  • Clearer first impression around the Dell Rewards value proposition
  • Better section pacing for a longer landing page experience
  • Stronger visual connection between Dell products and program benefits
  • More consistent enrollment path from top to bottom

Why it mattered

  • Users could understand the program without reading every detail
  • The page supported both quick scanners and users who wanted more information
  • The CTA system reduced the need to hunt for the next step
  • The mobile layout kept the experience clear and easy to follow

Learnings

This project reinforced how important hierarchy and pacing are when designing acquisition-focused landing pages with multiple benefits competing for attention. The challenge was not adding more information. It was organizing the experience so users could understand the value quickly without feeling overwhelmed.

It also reinforced the importance of designing modular systems that maintain clarity across desktop and mobile. The strongest decisions came from simplifying the page structure, creating a more intentional section flow, and guiding attention toward a clear enrollment path rather than relying on extra copy or visual noise.
